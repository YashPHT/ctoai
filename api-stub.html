Smart Academic Mentor - API Documentation

POST /api/chat

Request Format

{
  "message": "I want to create a task",
  "conversationHistory": [
    {
      "role": "user",
      "content": "I want to create a task"
    }
  ],
  "currentStep": null,
  "taskData": {}
}

Response Format (Task Creation Flow)

{
  "message": "Great! Let's create a new study task. What subject is this for?",
  "currentStep": "task_creation_description",
  "taskData": {
    "subject": "Mathematics"
  },
  "actions": [],
  "taskCreated": false
}

Response Format (Task Created)

{
  "message": "Perfect! I've created your task.",
  "currentStep": null,
  "taskData": {},
  "actions": [],
  "taskCreated": true,
  "task": {
    "title": "Complete Algebra Homework",
    "description": "Solve problems 1-20 from chapter 5",
    "subject": "Mathematics",
    "dueDate": "2024-10-25",
    "priority": "High",
    "completed": false
  },
  "motivational": "Great job! You're building excellent study habits! ðŸŽ‰"
}

Multi-Step Conversation Flow

The chatbot supports the following conversation steps:

  • task_creation_subject - Asking for subject
  • task_creation_description - Asking for description/details
  • task_creation_duedate - Asking for due date
  • task_creation_priority - Asking for priority level
  • study_plan_goal - Asking for study plan goal
  • study_plan_subject - Asking for study plan subject

Implementation Notes

  • The chatbot gracefully handles API failures with offline mode
  • All conversation data is persisted in localStorage
  • Tasks are stored locally and can be toggled as complete/incomplete
  • Typing indicators appear during API communication
  • Smooth animations and transitions throughout
  • Full keyboard navigation and ARIA support
  • Dark mode with persistent theme preference
  • Responsive design for mobile, tablet, and desktop

Testing the Implementation

  1. Open index.html in your browser
  2. Click the "Chat" button in the header
  3. Try the quick action buttons or type a message
  4. Since the API endpoint doesn't exist yet, the chatbot will use offline mode
  5. Test task creation flow by clicking "Create Task"
  6. Toggle dark mode with the theme button
  7. Test responsive design by resizing the browser window
